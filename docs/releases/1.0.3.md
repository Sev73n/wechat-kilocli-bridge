# Version 1.0.3

**Release Date:** May 11, 2026

## Overview

Version 1.0.3 is a reliability patch for live Codex and OpenCode bridge sessions. It fixes two user-visible failure modes: Codex replies that were already written to the session log but never returned to WeChat, and OpenCode conversations that kept receiving WeChat messages in an old session after the user created a new one locally.

This release also adds a WeChat-side `/new` workflow for OpenCode and tightens OpenCode reply cleanup so prompt echoes and inline reasoning text do not leak back to WeChat.

## Fixes

### Codex final reply recovery

- Codex session logs are now polled while a WeChat turn still needs completion recovery, even if the bridge-side app-server reference or connection becomes unhealthy.
- `task_complete` entries that already exist in `.codex/sessions/*.jsonl` can now produce the missing `final_reply` instead of leaving WeChat without an answer.
- `bridge.log` records `final_reply:` events and marks final-reply send failures with `context=final_reply`, making diagnostics clearer.

### OpenCode new session following

- WeChat now supports `/new` and `/new-session` for OpenCode sessions.
- The bridge forwards new-session requests through the local companion proxy, local companion IPC, runtime host, and OpenCode SDK.
- Local OpenCode CLI session changes are followed more reliably, including `session.new`, `/new`, `tui.session.select`, `command.executed`, `session.created`, and unscoped companion `global-event` session creation.
- The OpenCode adapter now accepts common session event shapes such as `sessionID`, `sessionId`, `properties.session.id`, and `properties.info.id`.
- When a local OpenCode session switch is detected, the bridge clears stale turn state so subsequent WeChat messages go to the new shared session.

### OpenCode prompt echo and reply cleanup

- WeChat-origin OpenCode prompts are marked before `promptAsync()` returns, preventing early SDK echo events from being treated as local terminal input.
- Recent WeChat prompt matching keeps prompt echoes out of visible output and final summaries.
- Inline OpenCode reasoning prefixes such as `The user... I should...` are stripped while preserving the actual answer.

## Upgrade Notes

Upgrade with:

```bash
npm install -g @unlinearity/cli-wechat-bridge@latest
```

After upgrading, restart any running bridge and companion terminals so they load the 1.0.3 runtime.

For OpenCode, use either:

```text
/new
```

from WeChat, or create a new session in the local OpenCode CLI. In both cases, subsequent WeChat messages should follow the new session.
