# Version 1.0.2

**Release Date:** May 11, 2026

## Overview

Version 1.0.2 is a focused npm patch for first-run WeChat authentication and expired iLink sessions. It makes the globally installed commands behave like a normal end-user CLI: if WeChat credentials are missing or no longer accepted by the server, the command opens the QR login flow in the foreground instead of failing later inside the hidden bridge process.

This release is especially important for users who install with:

```bash
npm install -g @unlinearity/cli-wechat-bridge
```

## Fixes

### Foreground login for global starter commands

- `wechat-codex-start`, `wechat-claude-start`, and `wechat-opencode-start` now check WeChat credentials before opening the visible companion.
- Missing credentials trigger the QR login flow in the current terminal, so users can scan and continue without knowing about repo-local setup commands.
- Expired saved credentials are detected before bridge runtime startup, avoiding repeated long-poll error output.

### Expired iLink session recovery

- The bridge now recognizes `errcode=-14` / `session timeout` as an authentication/session problem.
- Stale sync cursors are cleared and retried once.
- If the session is still expired, the error is treated as fatal auth instead of retryable network failure.
- `wechat-setup` is available as a global relogin command.

### Legacy state migration cleanup

- Repo-local legacy credentials under `~/.claude/channels/wechat` are only migrated when the global channel state is truly empty.
- If a user deletes the global `account.json` to test first-run setup, the old repo-local account is no longer silently copied back.
- Successful login clears stale `sync_buf.txt` and `context_tokens.json` so old session state does not leak into a new login.

### Clearer setup output

- The successful login message now focuses on npm global commands.
- Repo-local `bun run ...` guidance was removed from the normal setup success text.
- Historical `/pair` wording was removed from the success text.
- QR code output is written directly to stdout so it is not polluted by launcher log prefixes.

## Upgrade Notes

Upgrade with:

```bash
npm install -g @unlinearity/cli-wechat-bridge@latest
```

If a previous login is expired, run one of the normal starter commands and scan the QR code when prompted:

```bash
wechat-codex-start
```

Or force a relogin manually:

```bash
wechat-setup
```

No manual migration is required.
