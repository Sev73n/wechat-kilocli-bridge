# 版本 1.0.2

**发布日期：** 2026 年 5 月 11 日

## 概览

1.0.2 是一个面向 npm 全局安装体验的补丁版本，重点修复微信首次鉴权和 iLink 会话过期后的恢复流程。现在，全局安装后的命令会像普通用户 CLI 一样工作：如果缺少微信凭据，或已保存凭据被服务端判定过期，程序会在当前终端直接进入二维码登录，而不是等到隐藏的 bridge 进程里反复报错。

这个版本主要面向通过以下方式安装的用户：

```bash
npm install -g @unlinearity/cli-wechat-bridge
```

## 修复内容

### 全局 starter 命令前台登录

- `wechat-codex-start`、`wechat-claude-start` 和 `wechat-opencode-start` 会在打开可见 companion 前先检查微信凭据。
- 如果凭据不存在，会在当前终端显示二维码，让用户扫码后继续，不再要求用户知道仓库本地的 setup 命令。
- 如果已保存凭据过期，会在 bridge runtime 启动前识别出来，避免进入长轮询反复报错。

### iLink 会话过期恢复

- bridge 现在会把 `errcode=-14` / `session timeout` 识别为鉴权或会话问题。
- 程序会先清理旧的同步游标并重试一次。
- 如果服务端仍然返回会话过期，该错误会被视为不可重试的鉴权错误，而不是普通网络错误。
- 新增全局命令 `wechat-setup`，用于用户主动重新扫码登录。

### 清理旧状态迁移问题

- 仓库本地 legacy 凭据 `~/.claude/channels/wechat` 只会在全局状态目录真正为空时迁移。
- 如果用户删除全局 `account.json` 来测试首次运行流程，旧的 repo-local account 不会再被静默复制回来。
- 登录成功后会清理旧的 `sync_buf.txt` 和 `context_tokens.json`，避免旧会话状态污染新登录。

### 更清晰的 setup 输出

- 登录成功后的提示现在只面向 npm 全局命令。
- 普通 setup 成功提示中不再输出 repo-local 的 `bun run ...` 指南。
- 登录成功提示中移除了历史遗留的 `/pair` 表达。
- 二维码直接写入 stdout，避免被 launcher 日志前缀污染。

## 升级说明

使用以下命令升级：

```bash
npm install -g @unlinearity/cli-wechat-bridge@latest
```

如果旧登录已过期，直接运行常用 starter 命令并按提示扫码：

```bash
wechat-codex-start
```

也可以手动强制重新登录：

```bash
wechat-setup
```

不需要手动迁移配置。
